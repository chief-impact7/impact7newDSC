# 로그북 학년 필터 (초/중/고 다중 선택)

- 작성일: 2026-05-31
- 범위: impact7newDSC 로그북(`src/dashboard/`) 단독. 크로스앱 아님.

## 배경 / 목표

로그북 상단 필터에 소속·반만 있어 학년 단위로 좁혀 볼 수 없다. 소속과 반 사이에
**초등/중등/고등 버튼 3개**를 추가하고, 각 버튼의 드롭다운에서 학년을 **다중 선택**해
해당 학년 학생만 로그북에 표시한다.

## 요구사항 (확정)

- **위치**: 소속 `<select>`와 반 `<select>` 사이 (`App.jsx` 301↔303)
- **버튼**: 초등 / 중등 / 고등 3개
- **학년 옵션**: 초등(초4·초5·초6) / 중등(중1·중2·중3) / 고등(고1·고2·고3)
- **선택 동작**: 다중 선택(체크박스). 선택 즉시 필터 반영
- **결합**: 선택한 학년끼리는 OR(합집합), 소속·반 필터와는 AND
- **미선택**: 아무 학년도 선택 안 하면 전체 표시(필터 없음)
- **버튼 표시**: 그룹에 선택된 학년이 있으면 버튼 색 강조(active)만. 개수·칩 없음
- **드롭다운**: 한 번에 하나만 열림, 바깥 클릭 시 닫힘
- **진단평가 섹션 제외**: 학년 필터는 일반 그룹(정규/내신/자유/특강/비정규)에만 적용.
  진단평가(temp_attendance)는 항상 전체 표시

## 비목표 (YAGNI)

- 학년 옵션을 데이터 기반 동적 생성 (고정 옵션으로 충분)
- 선택 개수 배지·선택 학년 칩 표시
- 학년 필터 상태의 URL/로컬스토리지 영속화
- 초1~3 등 옵션 밖 학년 (학원 대상 아님)

## 설계

### 1. 상태 (`App.jsx`)

```jsx
const [gradeFilter, setGradeFilter] = useState(new Set()); // 예: {'초6','중1'}
```
`DailyLogBoard`에 `gradeFilter={gradeFilter}` prop으로 전달(기존 `branchFilter`/`classFilter` 옆).

### 2. 컴포넌트 (`src/dashboard/components/GradeFilter.jsx` 신규)

```jsx
const GRADE_GROUPS = [
  { level: '초등', short: '초', grades: ['4', '5', '6'] },
  { level: '중등', short: '중', grades: ['1', '2', '3'] },
  { level: '고등', short: '고', grades: ['1', '2', '3'] },
];
```
- props: `{ value: Set, onChange: (Set) => void }`
- 버튼 3개. 각 버튼 클릭 → 해당 그룹 드롭다운 토글. 열린 그룹 state 하나(`openGroup`)로 한 번에 하나만 열림
- 드롭다운: 그룹의 grades를 체크박스로. 키는 `short + grade`(예: `'초4'`). 체크 토글 시 `value` 복제해 추가/삭제 후 `onChange`
- 버튼 active: `grades.some(g => value.has(short + g))`
- 바깥 클릭 닫기: 컨테이너 `ref` + `useEffect`로 document mousedown 리스너, 바깥이면 `setOpenGroup(null)`

### 3. 필터 적용 (`DailyLogBoard.jsx` `buildLogData`)

- `buildLogData`에 `gradeFilter` 인자 추가. `DailyLogBoard`가 prop을 그대로 전달
- 학년 키 헬퍼:
  ```js
  const LEVEL_SHORT = { '초등': '초', '중등': '중', '고등': '고' };
  const gradeKey = (s) => (LEVEL_SHORT[s.level] || '') + (s.grade || '');
  ```
- **일반 학생 루프(`students.forEach`, ~272)에만** 적용:
  ```js
  if (gradeFilter?.size && !gradeFilter.has(gradeKey(student))) return;
  ```
- 진단평가(`tempAttendances`, ~225)에는 적용하지 않음 — 항상 전체

### 4. 렌더 (`App.jsx`)

소속 그룹 `</div>`(301) 다음, 반 그룹(303) 앞에:
```jsx
<GradeFilter value={gradeFilter} onChange={setGradeFilter} />
```
스타일은 기존 `dash-filter-group` 패턴을 따르고, 버튼/드롭다운/active 색은 `dashboard.css`에 추가.

## 검증 시나리오

- 초6만 선택 → 로그북 일반 그룹에 초등6학년만, 진단평가는 그대로 전체
- 초6 + 중1 선택 → 두 학년 학생 모두 표시(OR)
- 소속 2단지 + 초6 선택 → 2단지의 초6만(AND)
- 전체 해제 → 전체 표시
- 드롭다운 열고 바깥 클릭 → 닫힘. 다른 버튼 클릭 → 이전 드롭다운 닫히고 새 것 열림
