## 이슈 1: 소속 10단지 오표시 (branchFromClassNumber가 내신 csKey 오인)

**위치:** `app.js`
- `branchFromClassNumber` (248-254): `num` 첫 글자 `'1'→2단지`, `'2'→10단지`
- `branchFromStudent` (256): `s.branch || (... branchFromClassNumber(enrollments[0].class_number))`
- `activeBranchesFromStudent` (333-341): `relevantEnrollments(s)` 각 enrollment의 `class_number`를 `branchFromClassNumber`로 파싱, `set`이 비어야만 `s.branch` 사용
- 소속 표시 호출: 1498행, **1894행** (`activeBranchesFromStudent(...).join(', ') || branchFromStudent(...)`)
- `relevantEnrollments` (327-329) → `getActiveEnrollments` → `applyNaesinFreeDerivation`(311-320): **내신 기간 중 정규를 숨기고 내신으로 치환**

**원인:** 내신 기간 중 활성 enrollment는 내신 하나뿐이고 그 `class_number`는 정규 반번호가 아니라 **csKey 문자열 `"2단지마포고1A"`**. `branchFromClassNumber`가 첫 글자 `'2'`를 반번호 규칙(`'2'`=10단지)으로 해석 → **실제 2단지인데 10단지로 뒤집힘**.

**설계 의도 (사용자 확인):** 내신 기간 동안 소속은 **내신 듣는 단지로 표시**가 맞음. 단 학생 문서의 `branch` 필드는 정규 기준으로 **보존**(내신 배정이 branch를 안 건드림 — 표시만 파생). "정규 10단지 + 내신 2단지" 학생도 내신기간엔 2단지 표시가 정답.

**수정안:** `branchFromClassNumber`가 csKey 접두를 먼저 인식.
```js
const branchFromClassNumber = (num) => {
    const c = (num || '').trim();
    if (c.startsWith('10단지')) return '10단지';   // 내신 csKey
    if (c.startsWith('2단지')) return '2단지';       // 내신 csKey
    const first = c[0];                              // 정규 반번호
    if (first === '1') return '2단지';
    if (first === '2') return '10단지';
    return '';
};
```
결과: 심예율(내신 2단지)→2단지, 정규10+내신2 학생→내신기간 2단지·평소 10단지. branch 필드는 그대로 보존되어 표시만 파생.

**확인 필요:** csKey 접두 형식이 항상 `"2단지"`/`"10단지"`인지(다른 단지명 추가 가능성). `buildNaesinCsKey`의 branch 인코딩과 일치하는지 점검. newDSC는 `branchFromStudent`(branch 필드 우선)만 써서 이 버그 없음 — **impact7DB 한정**.

## 이슈 2: 퇴원 학생 수업추가 시 status 무단 재원 전환 (가드 부재)

**증상:** 03/12 퇴원 → 03/13 수업추가 과정에서 status가 가드 없이 재원으로 풀림. 퇴원생이 재원으로 바뀌고 history엔 '재등원'이 명시되지 않음.

→ 저장 함수 위치 확인 후 가드 추가 필요(impact7DB 세션에서 특정).
