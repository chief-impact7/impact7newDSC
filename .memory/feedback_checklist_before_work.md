---
name: 작업 전 체크리스트 필수
description: 다단계 작업을 시작하기 전에 항상 사용자가 볼 수 있는 체크리스트(TaskCreate)를 먼저 만들고 승인받은 뒤 진행
type: feedback
---

비단순 작업(3단계 이상, 여러 파일 수정, 환경 셋업 등)을 시작하기 전에 반드시 TaskCreate로 체크리스트를 만들고 사용자에게 보여준 뒤 진행한다. 작업 중에는 매 단계마다 TaskUpdate로 진행 상황을 갱신한다.

**Why:** 사용자가 1인 개발자이고 복잡한 작업이 길게 이어지면 무엇이 어디까지 진행됐는지 헷갈린다. "내가 자꾸 내용을 헷갈려" 라는 명시적 피드백이 있었다 (2026-04-14, module-splitter 2차 분리 직후). 체크리스트가 있으면 (1) 작업 범위가 사전에 명확해지고 (2) 진행률을 시각적으로 확인 가능하고 (3) 중단 후 재개 시 컨텍스트 복원이 쉽다.

**How to apply:**
- 트리거: 사용자가 다단계 작업(셋업/마이그레이션/리팩터/여러 파일 수정 등)을 요청하면 코드를 건드리기 전에 TaskCreate부터.
- 단순 1~2단계 작업(예: "이 한 줄 고쳐줘", "커밋해줘")은 예외 — 체크리스트 없이 바로 진행.
- 체크리스트 작성 후에는 사용자에게 "이대로 진행할까요?" 확인 후 시작.
- 각 task 시작 시 TaskUpdate(status: in_progress), 완료 시 TaskUpdate(status: completed).
- 작업 도중 새로운 step이 발견되면 TaskCreate로 즉시 추가.
- 사용자가 "건너뛰기" 같은 단축 지시를 하면 검증 step을 자동 skip하되 체크리스트에서는 그 task를 completed로 마무리 (어떤 step이 실제 검증됐는지/안 됐는지 사후 추적 가능하게).
