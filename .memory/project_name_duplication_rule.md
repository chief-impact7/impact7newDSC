description: 신규/재등원 학생 등록 시 동명이인 처리 규칙 - impact7DB 프로젝트에서 구현 필요
type: project
---

학생 이름 입력 시 중복 검증 규칙:

1. 동일한 이름의 **재원/휴원** 학생이 있는 경우 → 이름 뒤에 숫자를 붙이도록 알림, 숫자 붙인 이름도 있으면 계속 차단
2. 동일한 이름이 **퇴원/비재원**에만 있는 경우 → 입력 허용

**Why:** 동명이인 구분을 위해 이름 뒤 숫자를 붙이는 관례가 있으나, 입력 시 강제하지 않아 중복 발생. DSC 비정규 목록에서 구분 불가 문제 발생.

**How to apply:** impact7DB 프로젝트의 학생 등록/재등원 기능에서 구현. DSC는 진단평가(비등록)만 입력하므로 해당 없음.
